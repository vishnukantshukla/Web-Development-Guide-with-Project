CSS Layout -the Position property

The position property specifies the type of positioning method used for an element (static, relative, fixed, absolute, or sticky).

There are five different position values:

  • static
  • relative
  • fixed
  • absolute
  • sticky

Elements are then positioned using the top, bottom, left, and right properties. However, these properties will not work unless the position property is set first. They also work differently depending on the position value.

position: static;

HTML elements are positioned static by default. Static positioned elements are not affected by the top, bottom, left, and right properties.

An element with position: static; is not positioned in any special way; it is always positioned according to the normal flow of the page.

position: relative;

An element with position: relative; is positioned relative to its normal position. Setting the top, right, bottom, and left properties of a relatively-positioned element will cause it to be adjusted away from its normal position. Other content will not be adjusted to fit into any gap left by the element.

This is often used to create subtle adjustments to an element's position without affecting the layout of other elements. You can use positive or negative values for the top, right, bottom, and left properties to move the element in various directions.

position: fixed;

An element with position: fixed; is positioned relative to the viewport, meaning it remains in the same place even when the page is scrolled. This is commonly used for elements like navigation bars, headers, or sidebars that you want to remain visible as the user scrolls down the page.

By setting the top, right, bottom, and left properties, you can precisely position the fixed element on the screen. Remember that a fixed element won't leave a gap in the layout, so be cautious when using this property.

position: absolute;

An element with position: absolute; is positioned relative to its nearest positioned ancestor. If no ancestor is positioned, it's positioned relative to the body. Absolute positioning is often used to create overlays, tooltips, or other elements that should appear precisely at a specific location in relation to another element.

By combining position: absolute; with the top, right, bottom, and left properties, you can control the exact placement of the element. Keep in mind that absolute positioned elements are removed from the normal document flow, so they can overlap other elements.

position: sticky;

An element with position: sticky; behaves like position: relative; until it reaches a certain scroll position, then it "sticks" in place like position: fixed;. This is useful for creating headers or navigation bars that remain visible as long as they're on the screen.

Use the top, right, bottom, or left properties along with a value to define when the element becomes sticky. For instance, top: 0; would keep the element stuck to the top of the viewport when it reaches the top of the container.

position: relative;

An element with position: relative; is often used when you want to make small adjustments to the positioning of an element without disturbing the flow of surrounding content. You can use the top, right, bottom, and left properties to offset the element from its original position. This is useful for creating effects like slightly shifting an element upwards or to the right without impacting other elements.

position: fixed;

position: fixed; is commonly employed for creating persistent elements, such as navigation menus or call-to-action buttons, that remain visible as the user scrolls down the page. By setting the top, right, bottom, and left properties, you can determine the exact position of the fixed element. Keep in mind that fixed elements are not affected by scrolling and do not create gaps in the layout, so ensure they don't obscure other important content.

position: absolute;

position: absolute; is used for creating elements that need to be precisely positioned in relation to another element on the page. This is useful for creating tooltips, modals, or overlays. The key aspect of position: absolute; is that it's positioned relative to its closest positioned ancestor. If there are no positioned ancestors, the element is positioned relative to the document body. You can use the top, right, bottom, and left properties to control its position and appearance.

position: sticky;

The position: sticky; property is particularly handy for creating interactive and engaging user interfaces. It combines aspects of both position: relative; and position: fixed;. An element with position: sticky; behaves as if it's relatively positioned until the user scrolls past a specified offset, at which point it "sticks" to a fixed position within its containing element or the viewport. This can be utilized for creating elements like headers that remain visible until they're scrolled out of view, enhancing navigation and user experience.
